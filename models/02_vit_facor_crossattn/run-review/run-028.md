## Two Critical Issues Found

### 1. Validation accuracy stuck at 50.0% — hardcoded threshold

**Root cause**: `ViTFaCoRROCmTrainer.validate()` calls `evaluate_model()` with default `threshold=0.5`. The model outputs cosine similarities mapped to [0,1] via `(cos_sim + 1) / 2`. With contrastive loss, scores cluster in a narrow band where the fixed 0.5 threshold classifies everything as one class.

**Evidence**: AUC is 0.84+ (model CAN discriminate), but accuracy is exactly 50.0% on a balanced set (5695 pos / 5695 neg) — meaning all predictions land on the same side of the 0.5 threshold.

**Fix**: Override `validate()` to call `find_optimal_threshold()` before computing accuracy. This is what `evaluate_with_validation_threshold()` does post-training, but it was never used during training.

### 2. Overfitting after ViT unfreeze — no LR decay

**Root cause**: After unfreezing last 2 ViT blocks at epoch 4, the LR stays constant at 1e-5 with no scheduler. On a 166K-pair dataset with unfrozen backbone blocks, this is too aggressive — the model overfits the training set (loss drops from 0.0044 to 0.0007) while val AUC declines from 0.8715 to 0.8226.

**Evidence**:
- Train loss monotonically decreasing (0.006 → 0.0007)
- Val AUC peaked at epoch 5 (immediately after unfreeze), then declining for 19 straight epochs
- Epoch time jumped from ~1619s to ~1958s at epoch 5 (more params to train), confirming unfreeze happened

**Fix**: Use cosine scheduler to decay LR after unfreeze. The KinFaceW run 23 didn't need this because the dataset was tiny (~2K pairs) — overfitting took longer to manifest.

## Safeguards Needed

This run wasted ~13 hours before being manually killed. The existing early stopping (patience=30 on AUC) would have eventually stopped at epoch 35, but that's still 18+ hours for a clearly failing run. Need:
1. **Degenerate accuracy detector**: if accuracy == 50.0% for N consecutive epochs, warn and/or stop
2. **Sustained decline detector**: if monitored metric declines for M consecutive epochs from peak, trigger early stop
3. **Minimum improvement gate**: if best metric hasn't improved by X% after the first K epochs, stop early

## Key Takeaway

The FIW dataset is ~80x larger than KinFaceW. The run 23 recipe (designed for ~2K pairs) cannot be directly transferred — it needs LR scheduling and the evaluation pipeline needs threshold-aware validation during training, not just post-training.
