## Changes from Run 028
1. **Cosine scheduler** (was: none) — prevents overfitting after unfreeze
2. **Threshold-aware validation** — `validate()` now calls `find_optimal_threshold()` instead of hardcoded 0.5
3. **Safeguards added** — auto-stop on 10 consecutive epochs of AUC decline
4. **Negative ratio 2:1** (was: 4:1) — faster epochs, less overfitting risk
5. **Dropout 0.15** (was: 0.1) — more regularization

## Analysis

### What Worked
1. **Cosine scheduler prevented catastrophic overfitting** — run 028 degraded AUC from 0.87 to 0.82 with constant LR; run 030 held better (0.865 → 0.841 at stop)
2. **Threshold fix gave real accuracy** — 77.3% vs fake 50% in run 028
3. **Safeguard saved ~10 hours** — auto-stopped at epoch 19 instead of running to 50
4. **Per-relation metrics are informative** — sibling relations (sibs/bb/ss) are easiest; grandparent (gmgd/gmgs) are hardest

### What's Still Wrong
1. **Overfitting after unfreeze persists** — AUC peaked at epoch 8 (4 epochs after unfreeze at epoch 4) and declined steadily. Cosine decay slowed but didn't prevent it.
2. **Low TAR@FAR** — at FAR=0.01, only 12% of kin pairs are accepted. The model isn't confident enough for high-security applications.
3. **Non-kin F1 = 0.000** — all non-kin samples are being evaluated together as one "relation" class with no positive labels, so F1 is undefined. Not a real problem, just a metric display artifact.
4. **Val/test AUC gap** (0.865 vs 0.822) — suggests some overfitting to val distribution, though the gap is acceptable.

### Next Steps to Try
1. **Delay unfreeze to epoch 8-10** — let the cross-attention head train longer before disturbing the ViT backbone
2. **Lower post-unfreeze LR** — use differential LR: 1e-6 for ViT blocks, 1e-5 for head
3. **More aggressive cosine decay** — set T_max shorter or use warmup-restart
4. **Skip freeze entirely** — FIW has enough data (99K pairs) to fine-tune from epoch 1 with a low LR (5e-6)
5. **Data augmentation** — add random erasing, mixup, or stronger color jitter for regularization
