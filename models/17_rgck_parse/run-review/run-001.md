# Run 001 — Planning doc — M17 PG-RGCK (parsing-guided per-image region boxes)

**Status:** Prepared 2026-06-16, not yet launched. Code validated on CPU:
`model.py` smoke test builds + forward with per-image boxes + static fallback OK
(trainable params **31,036,173 = M12 R011 exactly**, 0 added by the parsing
tokenizer); the parsing tool's box logic, key scheme, and `.npz` round-trip pass;
the dataset's `_load_boxes`/`_box_key` match the cache format; `train.py`/`test.py`
import clean with the new trainer + box-aware eval helpers present.

M17 R001 = the **M15 R001 recipe** (= M12 R011 champion) with **one change**: the
four anatomical region boxes (eyes, nose, mouth, jaw) are **per-image**, the tight
bounding boxes of a face-parsing network's masks, instead of the fixed
`DEFAULT_REGIONS_224` rectangles. It is the first model in the project whose
region geometry actually adapts per face.

## Hypothesis

The methodology names fixed boxes as a limitation; M13 tried to lift it but used
**canonical** template landmarks, which on pre-aligned FIW are fixed by
construction (M13 was *"already a fixed-box model"* and underperformed). The
5-point similarity alignment normalises only 4 DOF — face shape, jaw width, brow
position, cheek geometry, expression and residual pose still vary per face. A
**dense per-image parser** captures that residual geometry.

**Claim:** region boxes that track each face's parsed parts improve robustness and
the strict-FAR / hard-relation (grandparent) operating points over fixed boxes,
even if aggregate AUC is near the project ceiling.

## What changed vs M15 R001 (the only delta)

| | M15 R001 | **M17 R001** |
|---|---|---|
| Region boxes | fixed `DEFAULT_REGIONS_224` | **per-image** from face-parsing masks |
| Box source | hard-coded | offline `.npz` cache (parser not in training loop) |
| Trainable params | M12 R011 | **same** (0 added) |
| Geometric aug | flip + rotation | **off** (cached boxes must stay registered) |
| Embedding space | AdaFace (cosines valid) | **same** (bbox→ROI-Align→FC unchanged) |
| Everything else | — | identical (symmetric, cmp-only, rel-aux 0.05, hard-neg 30%, diff-LR, stage-4, IMG_SIZE=160, seed 42) |

## Recipe

```bash
# build the cache once (offline, CPU ok):
python tools/parse_faces_boxes.py --backend bisenet \
    --bisenet-repo <repo> --weights <79999_iter.pth> \
    --aligned-root "$ALIGNED_ROOT" --out "$REGION_BOX_CACHE"
# train:
REGION_BOX_CACHE=/path/fiw_region_boxes.npz bash models/17_rgck_parse/AMD/run_m17_r001.sh
```

## What to watch

1. **Parser fallback rate** (printed by the tool, per region). If high (parser
   fails to find eyes/mouth on many faces), the boxes are mostly the fixed
   DEFAULT → M17 collapses toward M15 and the test is weak. Want < ~10–15%.
2. **Sanity control:** run R001 once on a `--backend dummy` cache (all fixed
   boxes) — it MUST reproduce M15 R001 within noise. Any real-parser run is then
   compared against that, not just against M15's logged number.
3. **Val→test gap & per-relation:** the hypothesis predicts gains concentrated in
   grandparents and strict-FAR, not necessarily aggregate AUC.
4. **No geometric aug confound:** since aug is off, also compare against M15 run
   with `--no_geometric_aug` (or read the dummy-cache control) so the box effect
   isn't confounded with the dropped augmentation.

## Decision rules

- **Test AUC ≥ M15 R001 + 0.008, or clear grandparent/low-FAR gain at ≥ M15 AUC**
  → adaptive geometry helps; CV (5 folds) to confirm.
- **≈ M15 (within noise) with low parser fallback** → adaptive boxes don't beat
  fixed on pre-aligned FIW; the geometry question is answered (with a *real*
  parser, unlike M13). Report as a clean negative; stop the box-source axis.
- **< M15 with high fallback** → parser/grouping problem, not the idea: tune the
  parser (better backend, region grouping, min-area) before concluding.
- **dummy-cache run ≠ M15** → plumbing bug; fix before trusting any real run.

## Honest prior / risks

- M16 R001 hinted the ceiling is representational/data-bound; M13 showed leaving
  the AdaFace embedding space hurts. M17 stays in that space (low risk) but may
  not move aggregate AUC. Even a null result is publishable: it closes the
  "adaptive geometry" question that M13 left open.
- Dropping geometric aug is a (documented) recipe deviation forced by cached
  boxes; the dummy-cache control isolates it. The contour-aware mask-weighted
  variant is the planned next code step if bboxes underperform.
- The parser adds an external dependency (BiSeNet weights) and an offline
  preprocessing pass; it is cached, so it costs no training VRAM.
