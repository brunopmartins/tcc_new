# Run 002 — Planning doc — M16 FA-RGCK (family-invariance, fixed λ ramp)

**Status:** Prepared 2026-06-11, not yet launched. Code changes syntax-checked;
schedule verified. Same architecture as R001 (no model.py change).

R002 is the **fair test** R001 never delivered. R001 tied R011 but, per its own
caveats, the family-invariance hypothesis was **never applied at strength**: the
DANN λ ramp was calibrated for 100 epochs while the run safeguard-stopped at
~ep14, so the tested best-val checkpoint (ep3) saw λ≈0.15 (effective weight
≈0.015) — essentially R011 plus a 1.5 % nudge. R002 fixes the three issues the
R001 review flagged, with everything else held verbatim.

## What changed from R001 (only the test setup, not the idea)

| Issue (R001 caveat) | R001 | **R002** |
|---|---|---|
| λ ramp length | tied to `--epochs` = 100 → λ≈0.15 at ep3 | **`dann_max_epochs=6`** (decoupled) → λ≈0.68-0.85 at ep3-4 |
| Ramp steepness | γ=10 (very steep) | **γ=5** (gradual climb, lets the backbone adapt ep1-2) |
| Base family weight | 0.1 (N=1, inconclusive) | **0.3** (first real step of the planned sweep) |
| Loss decomposition | total only (couldn't answer "memorised?") | **per-term log**: `base(BCE+rel)`, `CE_family(raw)`, `weighted_fam` |

Actual per-epoch λ the trainer prints (λ=0 on epoch 1, ramps thereafter):

```
ep1 λ=0.000 | ep2 0.394 | ep3 0.682 | ep4 0.848 | ep5 0.931 | ep6+ ≈0.99
```

So the best-val checkpoint (historically ep3-4 in this stack) sees λ≈0.68-0.85
— it **actually experiences the treatment**, unlike R001.

Verbatim from R001/R011: symmetric forward, comparison-only fusion, relation aux
0.05, role-matched hard negs 30 %, single LR 1e-5, IMG_SIZE 224, batch 8 ×
grad-accum 4, seed 42. `model.py` is unchanged — only `train.py` (decoupled
`--dann_max_epochs`, per-term logging) and `run_pipeline.sh` (forwards the new

- `AMD/train.py`: new `--dann_max_epochs` (0 ⇒ fall back to `--epochs`); the

## What to watch

1. **`base(BCE+rel)` trajectory** — the headline diagnostic R001 lacked. If it
   stays high (~0.27) as λ climbs, the invariance pressure is genuinely
   constraining the backbone (the kinship path can no longer lean on family
   memorisation). If it collapses to ~0.01 anyway, the model memorised despite
   the pressure → the ceiling is representational/data-bound.
2. **`CE_family(raw)`** — should stay well above ~0 (collapse ⇒ family trivially
   recoverable ⇒ no pressure) and below chance ln(486)≈6.19.
3. **Val→test gap** — the hypothesis predicts it tightens below R011's −0.021 now
   that λ is real at the selected checkpoint.
4. **Val AUC under the ramp** — if Val AUC craters at ep2-3 as λ jumps, the ramp
   is too aggressive: lower `FAMILY_ADV_WEIGHT` to 0.2 or raise `DANN_MAX_EPOCHS`
   to 10 (slower climb).

## Decision rules

- **Test AUC ≥ 0.884** → beats the R011 CV-ensemble headline as a single model. CV.
- **0.876 ≤ AUC < 0.884 AND gap < −0.021** → mechanism works even if aggregate
  AUC is in-band; CV and report gap-closure + low-FAR.
- **`base(BCE+rel)` high + gap NOT tighter + AUC flat** → this is the R001
  conclusion ("ceiling is representational, not removable family-memorisation")
  but now *actually tested* at full strength → it becomes a supported finding,
  and the next investment shifts off the invariance axis (M15 tokenizer / backbone).
- **Val AUC collapses early** → ramp too aggressive (see watch-item 4).

## Honest prior

R002 removes the schedule confound but keeps the from-scratch DANN tension: the
val-AUC peak is early, and a ramping invariance term can degrade val AUC as it
grows, so best-val selection may still mildly favour lower-λ epochs. The gradual
γ=5 ramp mitigates this (λ is already ≈0.68 by ep3). If the from-scratch tension
proves limiting, the cleaner follow-up is warm-start from the R011 checkpoint +
constant λ (deferred: needs the R011 best.pt on the training machine). Either way
R002 is a genuine test of the hypothesis at the planned strength, which R001 was
not.
