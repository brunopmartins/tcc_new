# Run 001 — Planning doc — M16 FA-RGCK (family-adversarial domain generalisation)

**Status:** Implemented 2026-06-10, not yet launched. Architecture smoke-tested
on CPU (random init): builds, forward returns the 8-tuple (M12 7-tuple +
family dict) with `sym_extras` correctly at index 6, family logits `(B, num_fam)`,
backward through the GRL at λ=0.5 runs. Trainable params **31,314,760
(44.42 %)** = M12 R011 (31,036,173) + family adversary (278,587).

M16 R001 = the **M12 R011 recipe** + a **family-adversarial objective** (DANN,
each FIW family = a domain). It is the first model in the project to attack the
val→test gap *directly* rather than via shortcut removal / regularisation /
ensembling.

## Hypothesis

The CV-AUC ceiling (R006/R010/R011/R012 all 0.873–0.879; only the R011 ensemble
broke it, to 0.8839) is set by **residual family-identifiable features** in the
adapted backbone. R012 confirmed the model memorises training families by
~epoch 3.

**Claim:** explicitly removing family identity from the per-face representation —
via a gradient-reversal family discriminator — closes more of the val→test gap
than the indirect levers tried so far, lifting test AUC and (especially) low-FAR
on held-out families.

The objective is tightly linked to the binding metric: the adversary minimises
recoverability of family identity, which is exactly the quantity the val→test
gap measures.

## Recipe

```bash
bash models/16_fa_rgck/AMD/run_m16_r001.sh
#   UNFREEZE_LAST_STAGE=1 RELATION_AUX_WEIGHT=0.05 SYMMETRIC_FORWARD=1
#   COMPARISON_ONLY_FUSION=1 TRAIN_NEGATIVE_STRATEGY=relation_matched
#   HARD_NEGATIVE_RATIO=0.30 LEARNING_RATE=1e-5 (single LR)
#   FAMILY_ADV_WEIGHT=0.1 DANN_GAMMA=10 ADV_HIDDEN=256
#   IMG_SIZE=224 BATCH_SIZE=8 GRAD_ACCUM=4 SEED=42
```

DANN λ schedule: `λ = 2/(1+exp(−γ·p)) − 1`, `p = epoch/epochs`, γ=10. λ≈0 at
epoch 0 (≡ M12 R011), → ~1 by the final epochs.

## Changes from M12 R011

| Component | M12 R011 | **M16 R001** |
|---|---|---|
| Generalisation lever | shortcut removal (symmetric + cmp-only) + hard negs | **+ family-adversarial DANN** (gradient reversal on gA/gB) |
| New params | — | family discriminator (≈0.28 M) |
| LR | differential (stage4/output/head) | single 1e-5 (adversary adds a top-level module) |
| Loss | 0.5·(BCE_AB+BCE_BA)+0.05·CE_rel | + `family_adv_weight·CE_family` (GRL-reversed) |
| Floor | — | λ=0 ⇒ identical to R011 |

## What to watch (first real run — integration sanity)

1. **Train BCE descends cleanly** in epochs 1–5 (the family CE must not dominate
   — if it does, lower `family_adv_weight`). The progress bar shows the current λ.
2. **Family CE stays > 0** and does not collapse to 0 (collapse ⇒ the adversary
   trivially wins / features already invariant ⇒ no pressure) nor explode.
3. **Val→test gap** — the headline diagnostic. Hypothesis predicts it *shrinks*
   below R011's ~-0.021 as λ ramps. If the Val-AUC peak also holds/rises, the
   adversary is helping.
4. **Per-relation** — family-invariance should help most where memorisation hurt
   (grandparents); watch they don't collapse.

## Decision rules

- **Test AUC ≥ 0.884 (single run)**: beats the R011 CV-ensemble headline as a
  *single model* — strong. CV immediately.
- **0.876 ≤ Test AUC < 0.884 but val→test gap < R011's**: the mechanism is
  working even if aggregate AUC is in-band; CV and report the gap-closure +
  low-FAR as the contribution.
- **Test AUC < 0.876**: sweep `family_adv_weight ∈ {0.05, 0.3}` and consider a
  coarser family bucketing before concluding.
- **Train BCE stalls / Val AUC collapses early**: λ too aggressive — lower γ
  (slower ramp, e.g. 5) or `family_adv_weight`.

## Sweeps / variants (after R001)

1. **`FAMILY_ADV_WEIGHT ∈ {0.05, 0.3, 0.5}`** — the single most important knob.
2. **γ ∈ {5, 20}** — ramp speed of the invariance pressure.
3. **Adversary target** — currently gA/gB (contextualised globals). Variant:
   the raw pre-context AdaFace embeddings (more identity-laden) for a stronger
   invariance signal (needs exposing the raw tokens).
4. **CV-fold ensemble** of R001 folds (the move that took R011 → 0.8839).
5. **Stack with M15** (hi-res ROI tokenizer + family adversary) — combines the
   best tokenizer with the direct generalisation lever.

## Honest prior

Adversarial training is higher-variance than the proven recipes, and a 571-way
discriminator on long-tailed families is hard. The DANN λ=0 floor (≡ R011)
bounds the downside to "no movement". The win condition is a **measurably
tighter val→test gap** with held AUC — that alone is a strong, novel TCC result
("the first direct attack on the generalisation gap"), whether or not it beats
0.8839 outright. Even a null result is publishable: it would localise the
ceiling to the *representation* (frozen stages 1–3 + dataset difficulty) rather
than removable family-memorisation, closing the question the whole project has
circled.

**Verdict: tied with R011 on AUC/AP/F1 (within noise); worse on strict-FAR.** The
family-adversarial objective did NOT raise the absolute ceiling at λ=0.1.
**Decisive observation:** M16's train loss stayed ~0.90 (the adversary fought
memorisation) vs M14/M12 collapsing to ~0.01 — yet it hit the *same* test AUC.
Preventing family memorisation gave **no test gain** → the ~0.876 wall is
representational/data-bound, not an overfitting artifact. Future: sweep
`FAMILY_ADV_WEIGHT` (0.3+).
