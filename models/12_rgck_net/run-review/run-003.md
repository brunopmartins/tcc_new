## Context

R002 became the new project headline (Test AUC 0.8564) by combining
Phase 2 partial unfreeze with the M12 architecture (region tokens +
cross-region attention + sigmoid gating + BCE classifier head). The
proposal §38 specifies a Phase 4 experiment: add a **supervised
contrastive auxiliary loss** with small λ to organise the embedding
space alongside the BCE classifier.

R003 implements Phase 4 directly:

`L_total = L_bce(logit, label) + λ × L_supcon(gA_norm, gB_norm, label)`

with λ=0.05 (proposal §28 default conservative weight).

The hypothesis: explicit supervision over `(gA, gB)` similarity adds a
geometric regularisation that complements the BCE signal — pulling true
kin pairs together, pushing non-kin pairs apart, in addition to teaching
the classifier head to output the right logit.

Expected outcome: small lift in Val/Test AUC over R002 (proposal §28
explicitly warns "λ alto = aproximações artificiais", suggesting modest
gain at low λ).

**Actual outcome: net negative.** Val AUC peak slightly lower (0.9306
vs R002 0.9323), Test AUC -0.005 (0.8510 vs 0.8564). Per-class effects
are uneven — siblings improved (sibs +3.4, bb +1.7) but grandparent
classes regressed (gmgs -1.6, gfgs -1.0).

## Changes from Previous Run (R002)

| Parameter | R002 | **R003** | Rationale |
|---|---|---|---|
| `SUPCON_WEIGHT` | 0.0 (off) | **0.05** | Phase 4 of proposal §28 |
| `SUPCON_MARGIN` | (n/a) | 0.3 (default) | margin for negative pairs |
| All other knobs | — | identical | controlled single-knob test |

Implementation:

- `RGCKBCELoss` in train.py extended to optionally accept supcon weight + margin
- Model forward extended to return 5-tuple `(logit, weights, attn_map, gA_norm, gB_norm)` so the loss wrapper can access L2-normalised contextualised global tokens
- SupCon term is margin-style label-aware contrastive:
  - Positives (`label=1`): `(1 - cos)²`, pulling toward 1
  - Negatives (`label=0`): `max(0, cos - (1 - margin))²`, pushing below `1-margin = 0.7`
- Model config (best.pt) records `supcon_weight` and `supcon_margin` so `test.py` can rebuild identical state if needed

## Analysis

### What worked

- **Implementation was clean.** Adding the SupCon term required:
  - 1 extra forward output (model returns 5-tuple now)
  - ~25 lines in RGCKBCELoss for the supcon term
  - 2 CLI flags
  - Env vars in run_pipeline.sh
  - Trainer didn't need changes (RGCKBCELoss handles the tuple)
- **Training was stable**. SupCon λ=0.05 did not cause oscillation or
  collapse — the trajectory mirrors R002 closely. The model converged
  fine; it just didn't reach a higher peak.
- **Same-generation classes (sibs, bb) improved** by 1-3 pp. This is
  consistent with what the SupCon term physically does: pull positives
  together in embedding space. Siblings tend to have visually similar
  faces (same generation, often similar age), so the contrastive force
  helps.

### What failed

- **Net Test AUC regressed -0.005** vs R002. SupCon didn't help and
  marginally hurt.
- **All TAR@FAR levels regressed** by 1-2 pp. The model's ability to
  rank kin above non-kin at controlled FAR went down.
- **Cross-generation classes regressed** (grandparent pairs gmgs -1.6,
  gfgs -1.0). These pairs have large visual differences (age gap, life
  stage); forcing their embeddings close in space conflicts with the
  AdaFace identity prior.
- **Val→test gap widened** slightly (-0.076 → -0.080). SupCon pulled
  embeddings into a more val-friendly geometry that didn't transfer
  perfectly.

### Hypothesis test result

**"SupCon λ=0.05 auxiliary improves Test AUC over R002" — REJECTED.**

Effect size is small (-0.005 net) but consistent across multiple
metrics. The proposal §28 explicitly warned about this case: "Parentes
não são necessariamente visualmente muito próximos. Se a contrastive
loss for forte demais, pode forçar aproximações artificiais." Even at
the conservative λ=0.05, the term is enough to cause measurable harm
on cross-generation classes.

### Likely root cause

Three interacting factors:

1. **The BCE classifier head already learns a good kinship decision
   boundary.** Adding SupCon on the same data adds redundant supervision
   that mostly reinforces what BCE already learns, but introduces a
   slight bias toward visually-similar positives.

2. **AdaFace's identity prior conflicts with cross-generational
   kinship.** AdaFace was trained to push *different identities* apart;
   parent and child are different identities. Forcing their embeddings
   close (via SupCon for label=1 pairs) directly opposes the backbone's
   pretrain bias. The result is a tug-of-war that ends up degrading
   generalisation.

3. **The cross-region adapter already provides the "compare regions"
   inductive bias.** The contextualised tokens `(gA, gB)` are already
   shaped by attention to *correspond* across faces. Adding an explicit
   cosine constraint is redundant with this.

### Implication for future runs

- **R004 (Phase 5 relation-type aux head) is a more promising
  direction.** It would add a *different* signal (which relation
  category, not just kin/non-kin) without imposing an embedding-space
  constraint. Could specifically help grandparent classes.
- **R005 (Phase 6 hard negatives via relation_matched)** is risky
  given M11 v4's negative result on full-FT models. The frozen-low-stages
  setup might behave differently, but it's a gamble.
- **R006 (Phase 3 full FT)** almost certainly regressive based on M11 v4.
  Skip.
- Other directions outside the proposal's sequence:
  - ROI Align region tokenizer (proposal §15 Strategy 1) — architecture
    change for speed + possible small accuracy gain
  - Adjust SupCon λ down to 0.01 or 0.02 to see if the negative effect
    diminishes (low priority — even at 0.05 the regression is small)

**M12 R002 (Test AUC 0.8564) remains the project headline.**
