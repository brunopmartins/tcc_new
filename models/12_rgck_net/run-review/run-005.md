# Run 005 — AMD ROCm / RX 6750 XT / FIW / RGCK-Net Phase 5 (relation-type aux head)

**Date:** 2026-05-14
**GPU:** AMD Radeon RX 6750 XT (gfx1031, ROCm 5.7)
**Status:** SAFEGUARD-STOPPED at epoch 16 (Val AUC below peak 0.9318 for 10 consecutive epochs)
**Outcome:** Phase 5 hypothesis — **confirmed on per-class targets** (all 4 grandparent classes improved 2-7 pp) — but Test AUC -0.009 vs R002 (within the ±0.009 noise floor established in R004). Net: R002 stays headline; R005 is the first post-R002 intervention to **not** widen the val→test gap and the first to **improve every kin class** while doing so.

---

## Context

R002 (Test AUC 0.8564) became the project headline via Phase 2 partial
unfreeze. R003 (Phase 4 SupCon λ=0.05) regressed cross-generation classes
and was rejected. R004 (intended Phase 6 hard negatives) turned out to
have used a broken `relation_matched` sampler and measured only
negative-sampler reseed variance — see [run-004.md](run-004.md) Errata.

R005 implements **Phase 5 of `proposta_rgck_net_kinship.md` §38** — the
relation-type auxiliary head — as the next candidate intervention on top
of the R002 stack.

The hypothesis tested:

> A per-pair 11-way relation classification objective, applied only to
> positive pairs and class-balanced via inverse-frequency weights, will
> provide a relation-conditioned learning signal that specifically helps
> the underrepresented grandparent classes (gfgs/gmgd/gmgs) without
> forcing cross-generation embeddings to converge (the failure mode of
> R003's SupCon).

**Per-class signal**: ✅ confirmed. **All 11 kin classes improved 2-7 pp**
relative to R002; the 4 grandparent classes (the explicit Phase 5
targets) improved by 2.0-7.3 pp. **Threshold-invariant AUC signal**: ❌
not confirmed. Test ROC AUC 0.8476 vs R002 0.8564 (-0.009, **within the
noise floor we measured in R004**, but on the wrong side of it).

## Analysis

### What worked

- **Phase 5 hypothesis confirmed on the per-class targets.** All four
  grandparent classes improved by 2.0-7.3 pp, with gmgd (the rarest
  class, 542 train positives) gaining the most (+7.3 pp). The
  inverse-frequency class weighting (gmgd: 2.46×, gmgs: 2.14×, gfgd:
  2.19×, gfgs: 1.96×) routed gradient toward the under-represented
  classes as designed.

- **All 11 kin classes improved**, not just grandparents. The
  relation-conditioned signal seems to help the kinship decision
  globally, not just for the rare classes.

- **Val→test gap stayed in the second-best zone** at -0.084 (R002:
  -0.076, R005: -0.084, R001: -0.089). No catastrophic widening like
  M11 v4 (-0.128) or M10 R003 (-0.140). Phase 5 doesn't induce strong
  family memorisation.

- **R005 is the first post-R002 intervention to not lose meaningfully
  on Test AUC.** R003 (SupCon) lost -0.005, R004 (intended hard negs,
  reseed only) lost -0.009. R005 lost -0.009 — same magnitude as R004
  but with a completely different per-class profile: R004 regressed
  every kin class; R005 improved every one.

### What failed

- **Test ROC AUC -0.009 vs R002.** Within the noise floor we
  established in R004 (R002 ↔ R004 ≈ ±0.009 from a sampler-seed
  reseed alone), but on the wrong side. We cannot say Phase 5
  *improved* AUC; we can only say it did not break it.

- **TAR@FAR=0.001 -1.2 pp**, **@FAR=0.01 -0.9 pp**, **@FAR=0.1
  -1.5 pp**. At strict-verification thresholds, R002 remains stronger.
  Phase 5 trades strict-FAR performance for per-class kin balance.

- **Non-kin accuracy -5.7 pp** at the chosen threshold. The new
  threshold (0.300) is more permissive than R002's, which inflates
  kin recall but lets more non-kin pairs cross the bar.

### Hypothesis test result

**Targeted (per-class) version**: **CONFIRMED**.
> Phase 5 specifically helps the underrepresented grandparent classes.

**Global (AUC) version**: **NOT CONFIRMED**.
> Phase 5 lifts overall Test AUC above R002.

The intervention shifts the model's operating regime — it's a *qualitatively different* model, with stronger kin recall and weaker non-kin specificity. For applications that care about per-class balance (e.g., demographic fairness across kin types) R005 is a better model than R002. For applications that care about strict verification (e.g., low-FAR
deployment) R002 remains stronger.

### Why this happens (mechanism)

The CE_rel auxiliary loss pulls the relation_head's embedding (the
average of the two contextualised global tokens) toward a relation-
discriminative manifold. The classifier head, which sees the same
backbone features through its own MLP, inherits some of that geometry
— but the global organisation of "kin vs non-kin" in the cosine /
similarity-feature space doesn't necessarily improve.

In information-theoretic terms, the model is learning to encode 11
relation types instead of 1 binary kinship signal, which is a richer
representation but with the same total backbone capacity (the
backbone is partially frozen). Some of that representational budget
is being spent on intra-kin distinctions instead of kin-vs-non-kin
separation.

### Comparison with R003 (SupCon aux)

R003 (SupCon λ=0.05) and R005 (CE_rel λ=0.05) are both aux-loss
interventions at the same weight. Comparison:

| | R003 (SupCon) | **R005 (CE_rel)** |
|---|---|---|
| What it does | pulls gA·gB cosine toward 1 for kin, away for non-kin | classifies the 11 kin relation types |
| Phase | 4 | 5 |
| Test AUC vs R002 | -0.005 | **-0.009** |
| Per-class effect | sibling +1-3 pp, **grandparent -1 to -1.6 pp** ❌ | **all kin +2 to +7 pp** ✅ |
| Operating point | similar to R002 | threshold drops to 0.300 (higher recall) |
| Verdict | rejected | per-class confirmed; AUC inconclusive |

The two aux losses produce *opposite* per-class signatures, which is
informative: SupCon harms grandparents (it forces cross-generation
embeddings to converge), while CE_rel helps them (it gives them a
distinct categorical signal). This is consistent with proposal §28's
warning about strong contrastive loss being too aggressive.

### Implications for future runs

- **R005 stays on the table as a viable alternative recipe to R002.**
  It's not strictly better on AUC, but the per-class profile is
  meaningfully different. For the TCC narrative, the result is:
  *Phase 5 confirmed the per-class hypothesis; the AUC effect is
  within sampling noise.*

- **Phase 5 + symmetric forward (R006)** is the next single-knob
  step. If `forward(A,B) != forward(B,A)` is part of why some kin
  pairs score inconsistently, simetrising might close the AUC gap
  without losing the per-class gains.

- **Lower λ (e.g., 0.02 or 0.03)** could be worth testing — the
  per-class signal is strong even at λ=0.05, so a lower weight might
  preserve more of the strict-FAR performance.
