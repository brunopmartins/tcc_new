## Context

Run 001 showed a Val AUC of 0.9116 with Test AUC of 0.806 — a -0.105 val→test gap, the largest in the project. The hypothesis going in was **overfitting to the validation distribution**: DINOv2 features are rich, LoRA has minimal regularization, and head dropout is shallow at 0.1. R002 stress-tests that hypothesis directly.

Four changes, all in the regularization direction: (1) cut peak LR by half, (2) double head dropout, (3) enable LoRA dropout, (4) double the relation-CE weight to push the multitask signal harder on grandparent classes.

If the hypothesis is correct, R002 should close the gap (lower Val AUC, equal or higher Test AUC). If the gap stays, the cause is structural — most likely a difference in family distributions between val and test in the RFIW Track-I split.

## Analysis

### Hypothesis test result: REJECTED

The val→test gap of -0.106 is statistically indistinguishable from R001's -0.105. Stronger regularization did **not** close the gap — both Val AUC and Test AUC dropped by ~0.007 in parallel. This rules out "overfitting to the validation distribution" as the cause of the gap.

### What did change

1. **Per-relation accuracy became more uniform.** bb/ss/sibs gained +3-6 pp, gmgd +5 pp. The relation head with λ=0.4 spread classification confidence more evenly.
2. **Strict-threshold performance dropped sharply.** TAR@FAR=0.01 went from 0.152 → 0.095. R002 gives up the high-precision regime that was R001's signature win.
3. **Convergence got faster.** Peak at epoch 4 vs epoch 12 in R001. The model is hitting its ceiling earlier under stronger regularization, suggesting capacity is not the bottleneck.

### What R002 did **not** fix

1. **gfgs and gfgd remain near-chance.** 38.8% and 49.3% respectively. Doubling the relation-CE weight didn't move them. **The bottleneck on grandparent classes is not the multitask signal.**
2. **Val→test gap.** As stated above.

### Likely root cause (revised after R002)

Combining R001 and R002 evidence with the train-data analysis:

1. **The val→test gap is structural.** RFIW Track-I splits families disjointly between val and test. The val families likely have visual properties (sharper kin similarity, less age variation) that yield artificially high val AUC. Confirming this requires a multi-seed run or 5-fold CV.
2. **Grandparent failure is data starvation.** The four grandparent classes have only 542-682 training pairs each (1.6-2.1% of train), vs 5,109-5,349 for parent-child (15-16%). The model receives 6-9× fewer gradient updates teaching grandparent kinship.

---

## Next Step

**Stop iterating regularization.** R002 conclusively shows that the val→test gap is not closed by hyperparameter changes on the standard regularization axes (LR, dropout, multitask weight). Two diagnostic-first directions:

1. **Multi-seed sanity check** (cheapest, most diagnostic): rerun R001 config with seeds 0 and 7. If gap is consistent across seeds, it's the split.
2. **Class-balanced sampling** for the grandparent failure: enable a `WeightedRandomSampler` that inverts class frequency in train, keeping all R001 settings. Cheap to set up.

Avoid: more regularization variants on the same axes, relation-loss-weight increases beyond 0.4, hyperparameter ablations without addressing data first.
