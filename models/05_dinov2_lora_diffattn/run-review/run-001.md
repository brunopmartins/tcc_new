## Context

First training run for Model 05. The model is the design hypothesis itself: **DINOv2 self-supervised backbone (frozen) + LoRA adapters + differential cross-attention + auxiliary relation head**. Defaults from `run_pipeline.sh` only — no overrides were considered until baseline behavior was observed.

The project hypothesis going in: a frozen self-supervised backbone with LoRA fine-tuning would beat the fully fine-tuned ViT (M02) and hybrid CNN+ViT (M03) by sidestepping the overfitting these suffered after unfreezing — and the auxiliary relation head would resolve the grandparent-class gap that VLMs and parametric models both fail on.

## Analysis

### What worked

1. **Highest Val AUC of the project so far** (0.9116). DINOv2 + LoRA captured stronger discrimination on validation than M02's fine-tuned ViT (0.885) or M03's hybrid (0.885), with **20× fewer trainable parameters**.
2. **Best TAR@FAR=0.01 across all models** (0.152). M05 outperforms M02/M03 (~0.13) and M06 (~0.06) at strict thresholds — useful for high-precision regimes.
3. **Stable training.** No OOM, no NaN. Frozen backbone + LoRA confirmed as a good fit for 12 GB VRAM.

### What failed

1. **Val→test gap = -0.105.** Test AUC of 0.806 is 0.044 below M02/M03 (0.850) despite a higher Val AUC. The gap is much larger than M02's typical -0.03.
2. **Grandparent classes ~40-52%.** The auxiliary relation head with weight 0.2 did not solve the gfgs/gmgs/gfgd/gmgd classes — these fail at near-chance levels.
3. **gfgs is the worst class** at 39.8% — opposite of M02 R031 where gfgs was the best class (95.9%). Different failure mode, possibly threshold-driven.

### Hypothesized causes (to be tested in subsequent runs)

1. **Overfitting to validation family distribution** — DINOv2 features are richer, the LoRA + heads may be exploiting validation-specific shortcuts → R002 will test this with stronger regularization.
2. **Insufficient relation-CE signal** (λ=0.2) — could be raised to 0.4-0.5 to push the relation head harder → R002 will test this.
3. **Insufficient training data on grandparent classes** — to be diagnosed before running more architectural variants.
