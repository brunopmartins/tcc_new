**Date:** 2026-05-12

**Status:** STOPPED MANUALLY at epoch 19/100 after 4 epochs below ep-9 peak — Val AUC 0.8919 (ep 9), Test AUC **0.7982** (val→test gap **-0.094**)

---

## Context

This is the first run of M09. The hypothesis: porting the **SAI/CI³Former
"interaction-inside-the-backbone" idea** to the AdaFace IR-101 backbone
will help with kinship verification on held-out families. M10 R003
established that the same AdaFace backbone with cross-attention **only at
the top** (M02-style FaCoR) yields Val AUC 0.8875 but Test AUC 0.7478 — a
catastrophic -0.140 val→test gap caused by AdaFace's identity-cluster
pretrain memorising the training family pool.

M09's architectural hypothesis: inject **bidirectional FaCoR cross-attention
after stages 3 (14×14×256) AND stage 4 (7×7×512)** of IR-101, so the
two faces' tokens query each other *while* features are still being
extracted. This should let the backbone adapt away from pure identity
discrimination during fine-tune, reducing the memorisation that crippled
M10 R003 on held-out families.

Expected outcome: Test AUC ≥ M10 R003 (0.7478) with a smaller val→test
gap. Stretch: approach or beat M02 R031 (Test AUC 0.850).

Actual outcome: **partial success**. Test AUC 0.7982 (+0.050 over M10
R003), val→test gap -0.094 (~33% smaller than M10 R003's -0.140) —
confirming the architectural hypothesis is *directionally correct*. But
the gap is still ~3× worse than M02 R031's -0.031, and per-relation
collapse on grandparent classes persists (30-37 % vs M02 R031's 88-96 %).

## Analysis

### What worked

- **Multi-stage cross-attn unlocks one epoch earlier and 0.05 higher than
  M10 R003.** Val AUC at peak LR (ep 5): 0.8237 vs M10 R003's 0.7191.
  The single-epoch +0.111 jump confirms the stage-3 cross-attn was
  holding accumulated useful gradients through the warmup, releasing them
  cleanly when LR ramped.
- **Val→test gap reduced ~33 % vs M10 R003** (-0.094 vs -0.140). The
  architectural hypothesis — pair-aware processing inside the backbone
  reduces family memorization — is **directionally validated**.
- **Every kin relation improves**, with no exceptions. The improvement
  pattern is consistent: same-generation classes (bb, ss) gain ~10 pp,
  parent-child classes ~3-6 pp, grandparent classes mixed (+2 to +11).
- **Stable, healthy convergence.** No oscillation crisis like R002 of
  M10 (cosine_contrastive instability), no monotonic decline like M10
  R001 (SAM ensemble feature collapse). Train loss decreases monotonically
  by ~0.025/epoch, val plateaus naturally.

### What failed

- **Test AUC still well below M02 R031 (0.850 vs 0.7982 — gap -0.052).**
  The multi-stage architecture closes some of the gap but does not bring
  the AdaFace backbone up to M02 R031's ViT-B/16 ImageNet pretrain on
  held-out families.
- **Grandparent classes remain catastrophic** (30-37 % vs M02 R031's
  88-96 %). The improvement over M10 R003 here is real (+2 to +11 pp),
  but the absolute level is still well below random in some cases.
- **Threshold management quirk:** training was killed mid-pipeline, so
  best.pt was patched manually with model_config. Threshold defaulted
  to 0.500 instead of the val-optimal 0.100 — fixable in the trainer's
  save_checkpoint (write model_config + val_threshold *inside* the
  trainer, not in train.py's finaliser).

### Hypothesis test result

**"SAI-inspired multi-stage cross-attention reduces AdaFace's family
memorization on FIW" — supported.** The val→test gap drops from -0.140
(M10 R003) to -0.094 (M09 R001), a ~33 % reduction. Test AUC improves
+0.050. Per-relation accuracy improves uniformly. The architectural
direction is correct.

But the *magnitude* of improvement is not enough to surpass M02 R031.
Multi-stage cross-attn alone — without additional regularisation,
balanced sampling, or partial backbone freeze — does not close the
remaining gap to the project headline.

### Likely root cause of remaining gap

Two factors interacting:

1. **AdaFace's identity-cluster pretrain still dominates** even with
   pair-aware processing inside the backbone. The stage 3 + 4 cross-attn
   layers add ~3 M params but the IR-101 body (65 M params) still gets
   100 % gradient flow, and its first 16 ResBlocks (stages 1-2) are
   completely identity-discriminative. The pair signal injected later
   has to compete against that prior on every step.

2. **FIW class imbalance amplifies the problem.** Grandparent classes
   are ~10× rarer than parent-child. Random sampling means the model
   sees them ~10× less, and AdaFace's prior is ~10× more confidently
   wrong about their kinship. Multi-stage cross-attn helps marginally
   here (+2 to +11 pp) but cannot make up the data scarcity.

The val→test gap reduction (-0.094 from -0.140) is the cleanest signal
that multi-stage is doing useful work; the absolute test number being
below M02 R031 is the cleanest signal that more is needed.

## Next Step

**M09 R002: relation-balanced sampling + per-relation train metrics.**

The R001 result diagnoses the next bottleneck cleanly: grandparent
classes need direct intervention. Balanced sampling pushes them from
~1/14 of each batch to ~1/14 of each *class*, ~3× boost in effective
gradient signal. Per-relation metrics in the train phase (not only test)
will surface class-level regressions epoch-by-epoch, instead of
discovering them only post-test as in R001.
