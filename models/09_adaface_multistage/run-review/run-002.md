## Context

R001 established the M09 multi-stage architectural baseline (Val AUC
0.8919, Test AUC 0.7982) but left a clear failure mode: grandparent
classes collapsed to 30-37 % test accuracy while majority classes
held 50-65 %. The diagnosis was data-driven — FIW Track-I has
**~10× fewer pairs per grandparent class** (98-138) than per
parent-child class (~1000). Random sampling means the model sees
grandparent pairs roughly 10× less, and AdaFace's identity-cluster
pretrain memorises the few it does see.

R002's hypothesis: **`WeightedRandomSampler` that equalises relation
classes among positives will boost grandparent class generalisation
without hurting majority classes** — closing the val→test gap (R001's
-0.094) by forcing exposure to the rare classes during fine-tune.

A secondary contribution of R002: **per-relation metrics logged
per-epoch** in the trainer log line. This lets future runs detect
class-level regressions in real time instead of discovering them only
in post-test analysis.

Expected outcome: Test AUC ≥ 0.80 (modest improvement over R001's
0.7982), with grandparent classes climbing toward 50-60 %.

Actual outcome: **Test AUC = 0.7824 (-0.0158 vs R001)**, val→test gap
**widened** to -0.107. Only `gfgd` improved substantially (+19.5 pp); the
other 3 grandparent classes stayed within ±3 pp of R001, and all 7
majority classes regressed by 1-6 pp.

## Changes from Previous Run (R001)

Only two changes — both implemented within the project codebase:

| Parameter | R001 | **R002** | Rationale |
|---|---|---|---|
| Train DataLoader sampler | `shuffle=True` (random) | **`WeightedRandomSampler`** | Equalise positive class frequencies; rare classes (grandparent) get ~10× more relative exposure |
| Per-relation val metrics | Computed but not logged per-epoch | **Printed per-epoch** in trainer log | Real-time visibility of class-level regressions during training |

Sampler implementation (`models/09_adaface_multistage/AMD/train.py`):

- For each positive pair of relation `r`, weight = `1 / count(class r)`
- For each negative pair, weight = `(n_pos_classes × neg_ratio) / count(negatives)`
- Total positive weight = `n_pos_classes` (= 11)
- Total negative weight = `n_pos_classes × negative_ratio` (= 22 for 2:1 ratio)
- Effective sample ratio ≈ desired neg ratio, maintained

Per-rel logging in trainer (`models/shared/AMD/trainer.py`):

- Added line `per-rel | bb=X fd=X fs=X gfgd=X ...` after the main epoch
  log, sourced from `val_metrics["per_relation"][rel]["accuracy"]`
- `non-kin` class excluded from the line (it's the negative pool, F1=0
  by definition)

Everything else (LR, loss, batch_size, grad_accum, architecture) is
identical to R001.

## Analysis

### What worked

- **`gfgd` (granddaughter-grandfather) improved +19.5 pp** — the largest
  per-class gain of any change in the project so far on this relation.
  This was the smallest absolute count (N=138 test) but where the
  balanced sampler had the most leverage.
- **Per-relation train metrics infrastructure** — now wired into the
  shared trainer and visible per-epoch. Real-time visibility for all
  future runs. Reusable by every model that uses the shared trainer
  (M02, M03, M05, M06, M08, M10).
- **Healthy convergence shape** — no oscillation crisis, no instability.
  Same unlock pattern as R001 (ep 5 peak LR), same plateau-with-mild-
  oscillation post-unlock. Sampler did not break the optimisation.

### What failed

- **Test AUC -0.0158 vs R001 (0.7824 vs 0.7982)** — net regression on
  the headline metric. The balanced sampler bought a single grandparent
  class gain (+19.5 pp gfgd) at the cost of all 7 majority classes
  (-1.3 to -6.1 pp). On 13,425 test pairs total, the majority classes
  dominate the global AUC, so the net effect is negative.
- **Val→test gap widened** to -0.107 from R001's -0.094 (-13 % relative
  worse). The balanced sampler did *not* close the family memorization
  gap; it appears to have shifted memorisation onto the over-sampled
  rare-class families.
- **Per-rel val numbers in early epochs (90-95 % on grandparent classes)
  did not translate to test.** Test per-rel grandparent stayed at 33-50 %.
  This confirms that high val per-rel under heavy oversampling is an
  artifact of training distribution, not real cross-family learning.

### Hypothesis test result

**"Balanced sampler boosts grandparent class generalisation by forcing
exposure" — partially supported (`gfgd` only), but the *net* hypothesis
(boost without hurting majority classes) is **rejected**.**

Three classes saw essentially no benefit (gfgs/gmgd/gmgs: +1.6 to +3.1 pp,
within noise margin). One class saw a real boost (`gfgd` +19.5 pp). All
seven majority classes regressed by a clear margin (-1.3 to -6.1 pp).

### Likely root cause

The balanced sampler effectively trains on a *different data distribution*
than the test distribution. FIW Track-I test is naturally imbalanced
(majority classes ~10× more frequent than grandparent). By forcing
training batches toward uniform class distribution, the model:

1. Spends more capacity learning grandparent-specific patterns (which on
   AdaFace's identity-cluster prior translates to memorising the specific
   grandparent families seen in train).
2. Spends less capacity learning the broad majority-class patterns,
   regressing on bb/fs/ms/md/ss/sibs/fd test accuracy.
3. Reaches similar Val AUC (-0.0025) because val also has the same
   majority-skew as test, and ranking is preserved within each class
   pool — but per-class calibration shifted.

The val→test gap *widening* suggests that the oversampled rare classes
are now memorised harder than majority classes were in R001. Balanced
sampling didn't *spread* the memorisation evenly; it *concentrated* it
on the over-sampled classes.

### Implication for future runs

- Pure class-balanced sampling is rejected as a generalisation lever on
  this AdaFace setup.
- The interventions that should be tried next instead:
  - **Augmentation pack** (R003 was planned): variance in input pixels
    forces the model to learn invariances, not memorise pixel patterns.
    Operates orthogonally to class frequency.
  - **Partial backbone freeze** (R004 planned): freeze stages 1-2 of
    IR-101 so the low-level features (which are identity-discriminative
    by AdaFace pretrain) stay intact, only stages 3-4 + cross-attn + head
    adapt for kinship.
  - **Smaller peak LR** (R005 planned): gentler fine-tune reduces memorisation
    pressure regardless of class frequency.

The `gfgd` +19.5 pp gain is real and the only positive signal from R002.
A future variant could try **soft** rebalancing (e.g., interpolating
between random and balanced weights with α=0.3) to capture some of the
`gfgd` benefit without paying the full majority-class cost.
