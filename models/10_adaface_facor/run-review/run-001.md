**Date:** 2026-05-11

**Status:** STOPPED MANUALLY — SAM age ensemble caused monotonic val AUC decline; aborted at iter 110/16604 of epoch 5

---

## Context

This is the first run of M10. M10's design hypothesis is that AdaFace IR-101
(WebFace4M, face-discriminative pretrain) is a strictly better drop-in
backbone than M02's ImageNet ViT-B/16 under the same FaCoR cross-attention
recipe. M02 R031 set the project ceiling at Test ROC-AUC = 0.850; M10 was
expected to match or surpass that.

R001 layered a second axis on top: **SAM-based age augmentation** to add
generational invariance. The SAM preprocessor was run offline ahead of
this experiment, producing 151,260 aged variants (50,420 source × 3 target
ages of 8, 25, 70). The dataloader was modified to return `(B, 4, 3, H, W)`
per face — original + three aged siblings — and the model averaged the
resulting token grids and global embeddings (weighted: original 0.5,
each aged variant ≈ 0.167) **before** FaCoR cross-attention.

The intuition was: if the model sees the same person at multiple apparent
ages every step, it should learn an age-invariant representation, and
kinship verification across generation gaps (especially the rare
grandparent classes) should improve.

The data did not bear out the intuition.

## Analysis

### What worked

Pipeline mechanics (dataset, dataloader, model, ROCm path, AMP, gradient
accumulation, age-ensemble averaging) all executed without errors. AdaFace
checkpoint loaded cleanly (no missing/unexpected keys). VRAM stable around
5 GB at b=4 with the 4× backbone passes per sample. GPU util steady at
~99 %.

### What failed

**The age ensemble killed the kinship signal.** Four consecutive epochs
of monotonic Val AUC decline (0.6685 → 0.5718, −0.097 absolute) while
train loss collapsed (0.820 → 0.129, −84 %). The trajectory was so
unambiguous and so far from M02's epoch-1 behaviour (~0.78 Val AUC, then
monotonic *increase* during warmup) that continuing would have been
expensive noise.

### Hypothesis test result

**The "average aged tokens before cross-attention" design is rejected.**
The token-level ensemble forces the AdaFace IR-101 backbone to produce a
*single* feature map per face that fits the original *and* three aged
variants simultaneously. The local minimum it converges on is roughly:

> the model assigns nearly identical embeddings to every face, regardless
> of identity *or* kinship.

That produces low cosine_contrastive loss on most training pairs (because
positives are forced together and the loss for negatives is bounded by
the margin) while collapsing the per-sample variance that the val AUC
measures. The 50 % validation accuracy throughout is the smoking gun: the
classifier head cannot find *any* threshold that separates the validation
pairs above chance.

### Likely root cause

Kinship verification is not age-invariant — it is *partially*
age-conditional. A father-daughter pair at ages 40 and 10 shows kinship
*through* the age gap (eye shape, jaw line, bone structure that persist
through ageing). Averaging the father's adult features with synthetic
elderly and child versions of himself, then comparing to the same average
applied to his daughter, destroys exactly the cross-generational alignment
the model needs. Generational invariance done at the *feature average*
level is generational *erasure*.

A second contributing factor: SAM produces StyleGAN2-based crops that have
strong synthesis-domain texture. Forcing AdaFace (trained on natural
WebFace4M images) to encode these alongside real images may further
destabilise the backbone's pretrained statistics.

If SAM augmentation is reattempted in a later run, the design has to
change to a *per-sample-per-epoch* augmentation (pick one of {original,
age_8, age_25, age_70} at random per face per step), not a token-level
ensemble. That treats age as a noise/regularisation channel rather than
forcing invariance into the feature map.
