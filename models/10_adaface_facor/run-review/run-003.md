# Run 003 — AMD ROCm / RX 6750 XT / FIW / BCE classifier head — STABLE BUT OVERFITS

**Date:** 2026-05-12
**GPU:** AMD Radeon RX 6750 XT (gfx1031, ROCm 5.7)
**Status:** STOPPED MANUALLY at epoch 18 for early validation — Val AUC 0.8875 (ep 17), Test AUC **0.7478** (-0.140 gap)

---

## Context

R001 (SAM age ensemble) and R002 (cosine_contrastive on AdaFace IR-101) both
failed for different reasons — R001 collapsed kinship signal via feature-
level age averaging, R002 oscillated because cosine_contrastive doesn't pin
embedding orientation on a face-discriminative backbone. R003 was designed
to isolate the architectural axis from the loss axis by switching to a BCE
classifier head over `[emb1, emb2, diff, product]`. This decouples the
kinship decision (logit) from the embedding orientation (free to keep
AdaFace's identity cluster structure).

Expected outcome: a clean run that reaches the M02 R031 reference (Test AUC
≈ 0.85) or beats it.

Actual outcome: a clean run that **reaches an unprecedented Val AUC (0.8875)
for M10** but produces Test AUC = 0.7478, well below M02 R031's 0.850. The
val→test gap is -0.140, ~4.7× worse than M02's -0.030.

## Analysis

### What worked

- **BCE head fixed the cosine_contrastive instability.** No threshold
  flipping, no oscillation, monotonic Val AUC climb during warmup, stable
  plateau formation. The single-knob change from R002 was the right call.
- **Convergence dynamics are healthy.** Train loss saturates smoothly, val
  metrics improve in well-defined phases (warmup plateau → unlock →
  breakout → meso-plateau → minor peaks). Almost identical shape to M02
  R031's curve, just shifted.

### What failed

- **Catastrophic val→test generalisation gap (-0.140).** Val AUC 0.8875
  → Test AUC 0.7478. The model performs much worse on truly held-out
  family pairs than on the 15 % val slice carved from train families.
- **Per-relation collapse**, especially for grandparent classes. M02 R031
  produced 88–96 % accuracy across all relations; R003 sits at 20–58 %.
  The model is essentially producing one global score with no relation-
  conditional discrimination.

### Hypothesis test result

**The "AdaFace IR-101 + FaCoR + BCE" recipe runs to convergence cleanly
but does not transfer to FIW Track-I test data.** R003 establishes a
stable M10 reference point at Test AUC 0.7478 — below project best
(M02 R031 = 0.850), below M06 R001 (0.776), but above M08 R001 (0.693).

### Likely root cause

AdaFace IR-101's pretrained embeddings cluster faces by *identity*, not
by *family*. When BCE is applied with full backbone fine-tune, the model
learns to memorise *which families it has seen* — this works on val
(same family pool as train, just different pairs) but fails on test
(completely held-out families). M02 R031 didn't have this problem
because ViT-B/16 ImageNet had no built-in identity prior to overfit on.

In short: **AdaFace's discriminative pretrain is a liability, not an
asset, for kinship verification on held-out families.** This echoes the
M08 anti-kinship finding but through a different mechanism (overfit
rather than frozen-loss conflict).

## Next Step

**Move to M09 (multi-stage cross-attention).** M09 keeps the AdaFace
backbone but injects cross-attention *inside* the body (after stages 3
and 4) rather than only at the top. The hypothesis is that interleaving
pair-aware processing with feature extraction will let the backbone
adapt away from pure identity discrimination during fine-tuning.

If M09 also overfits, the path forward for M10 is:
- **R004:** strong regularisation (`DROPOUT=0.4`, `WEIGHT_DECAY=1e-3`)
  to combat val-pool memorisation
- **R005:** partial backbone freeze (only fine-tune last 2-3 stages)
- **R006:** smaller LR (peak 1e-6 or 2e-6) for gentler fine-tune
