## Context

First training run for Model 06. The model is a **non-parametric retrieval-augmented kinship verifier**: a frozen ViT encodes each pair, the pair signature is matched against a gallery of 33,207 positive training pairs, and the top-K retrieved supports feed a cross-attention module along with their relation labels.

Two project hypotheses being tested:

1. **Retrieval can substitute for backbone fine-tuning.** A frozen ViT + non-parametric memory should reach competitive Test AUC with ~10× fewer trainable params than M02/M03.
2. **Retrieval should help rare relation classes.** Giving the model direct access to known grandparent-grandchild pairs at inference time should fix the ~0% accuracy that VLMs and the ~88% accuracy of parametric models (M02 R031) get on those classes — but specifically *uniformly*, not by tuning thresholds.

R001 uses defaults from `run_pipeline.sh` only.

## Analysis

### What worked

1. **Project hypothesis 1 confirmed.** Retrieval substitutes for backbone fine-tuning at the cost of -0.07 AUC. With only 8.16M trainable params (vs 86M-176M for M02/M03), Test AUC reaches 0.776.
2. **Project hypothesis 2 confirmed.** Per-relation accuracy is **uniform** at 61-87% across all 11 classes — the only model in the project that doesn't fail catastrophically on grandparents. R001's 61% on gmgs is the highest among all models for that class. This validates the retrieval mechanism: giving the model direct access to grandparent-grandchild examples at inference time fixes what multitask supervision alone could not in M02/M03/M05.
3. **Stable training.** No OOM, smooth convergence.

### What didn't work

1. **Test AUC ceiling at 0.776.** Below the parametric models (0.850). The frozen encoder is the obvious bottleneck — DINOv2 features for kinship are still strong but not adapted to the task.
2. **Mild overfitting after epoch 8.** Train loss kept falling (1.061 → 0.888) while val AUC plateaued. The 8M head capacity is enough to overfit eventually.
3. **Strict-threshold performance is weak.** TAR@FAR=0.01 = 0.062 is the lowest among the trained models. The architecture struggles with high-precision regimes — likely because retrieval averages noise from non-kin neighbors.

### Key design observation

The val→test gap of -0.060 is large but not catastrophic. The model generalizes — just not as well as the parametric ones. The gallery is fixed across val and test (it's always the train-set positives), so any val-specific overfitting comes from the cross-attention learning gallery↔val correlations rather than the gallery itself.

## Next Step

Test whether scaling retrieval helps. R002 will swap the encoder to DINOv2 (richer features), double K to 64 (more retrieval context), and double the relation-CE weight to 0.30 (sharper multitask signal). Hypothesis: ceiling lifts toward 0.83-0.85 AUC.
