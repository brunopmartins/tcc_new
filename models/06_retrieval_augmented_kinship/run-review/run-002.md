## Context

R001 reached Test AUC=0.776 with frozen ImageNet ViT-B/16 + K=32 retrieval. R002 hypothesizes that **scaling the retrieval mechanism** lifts the ceiling: a richer self-supervised backbone (DINOv2) provides better facial features, doubling K provides more retrieval context, and a stronger relation-CE weight pushes the multitask signal harder.

Going in, the gallery↔val correlation was assumed to be a benign side-effect: the gallery is the train set, so a fixed gallery shouldn't introduce val-specific shortcuts. R002 directly tests whether richer features + more context lift this ceiling.

## Analysis

### Hypothesis test result: REJECTED, hard

Every directional change in R002 made test performance **worse**, not better:

1. **Backbone DINOv2 didn't help.** Val AUC peaked lower (0.8228 vs 0.8361). The richer self-supervised features didn't translate into better generalization.
2. **K=64 amplified the problem.** Doubling retrieval context didn't add discriminative signal — it added more correlated supports that the cross-attention could memorize.
3. **Relation-CE weight 0.30** — can't be attributed to the regression in isolation (no ablation), but doesn't appear to have helped.
4. **Threshold drift.** Val threshold rose from 0.40 to 0.55 because R002's score distribution is tighter. The shift hurt test recall more than it helped val precision.

### Likely root cause

The val→test gap **grew** from -0.060 to -0.092. This is the smoking gun: under richer features and more retrieval, the cross-attention finds more gallery↔val correlations to exploit. The gallery is the same train-set positives in both runs, so the change isn't gallery composition — it's the model's ability to fit gallery-specific patterns when given richer inputs.

This reframes the architecture's bottleneck. **The frozen encoder isn't the limiting factor.** Earlier intuition said "frozen → retrieval is the bottleneck → scale retrieval." R002 disproves that. The actual bottleneck is **regularization of the cross-attention against gallery memorization**.

### Per-relation: also worse

R001 had the most uniform per-relation accuracy of any model (61-87%). R002 collapsed to 41-67%. The relation-CE weight increase didn't even help the grandparent classes — they regressed alongside everything else.

---

## Next Step

**Stop scaling retrieval surface.** R002 is a strong negative result. The right axis is *what* gets retrieved, not *how much*:

1. **Hard-negative supports.** Mix in retrieved-but-non-kin pairs alongside the K positives. The cross-attention currently only sees "this is what kin looks like" — it never sees "this is what non-kin that retrieval picked looks like." Adding contrast forces discrimination.
2. **Return to ImageNet ViT-B/16** (R001's backbone) with all other R002 settings, to isolate which change drove the regression.
3. **Reduce K to 16** for an inverse-direction check — confirms that K alone correlates negatively with generalization.
4. **Gallery refresh with sub-sampling.** Refresh gallery each epoch with a random subset (e.g. 10K of 33K) to break gallery↔val correlation.

Avoid: more architecture additions before fixing the gallery/cross-attention regularization issue. LoRA is now saved for after this is resolved (would otherwise overlap M05).
