# M14 R001 — RGCK-Net + LoRA backbone

**Date:** 2026-06-10 · **GPU:** RX 6750 XT (gfx1031, ROCm 5.7)
**Recipe:** M12 R011 data+head stack (symmetric forward + comparison-only fusion
+ relation-aux 0.05 + role-matched hard negatives 30%) with the AdaFace backbone
adapted by **LoRA r16/α16 on stage 4 + output_layer** instead of the full unfreeze.
Single LR 1e-4, FixedPartition tokenizer @112. Trainable **5.95 M (8.37%)** vs
R012's 34.6 M.
**Status:** stopped at ep 22, finalized on best.pt (ep 5). Val AUC peaked **0.9017
@ep5** then stayed flat 0.890–0.900 (no val *collapse*) while train loss fell to
~0.13 (it *did* fit the train set). The safeguard (decline-based) never tripped, so
it was stopped + finalised manually. See caveats below.

## Verdict

**LoRA underperforms the full stage-4 unfreeze** — AUC 0.8700 is ~0.006–0.011
below the R011/R012 unfreeze line, and below on every metric. Provisional reading:
the ~0.9 M backbone adapters have less adaptation capacity than the 31 M unfreeze,
so it caps lower on test. **But N=1 with several confounds (below) — treat as
suggestive, not settled.**

## Caveats & limitations (added 2026-06-11)

**1. Val→test gap — it WIDENED, not shrank.** Peak Val AUC 0.9017 (ep5), Test
0.8700 → gap **−0.0317**, *wider* than R011's −0.021 and R012's −0.026. So LoRA
did **not** generalise better; it generalised **worse** on this run. This
contradicts the "low-rank = better regularisation" motivation and is the cleaner
way to state the negative (lower ceiling *and* a looser gap).

**2. Memorisation, decomposed — it DID memorise the train set.** Loss is
`0.5·(BCE_AB+BCE_BA) + 0.05·CE_rel` (no family term to inflate the total, unlike
M16). Epoch-mean train loss fell monotonically **0.64 → 0.13** (ep21) while Val AUC
peaked at ep5 then drifted down — i.e. BCE is small and the model fits the training
families. (An earlier note citing "~0.01" was a noisy single-batch value; the
epoch-mean floor is ~0.13.) So the earlier phrasing "LoRA did not memorise" is
**corrected**: train memorisation happened; what LoRA avoided was a *val collapse*
(val held ~0.89–0.90 rather than crashing). Memorising-but-stable still gave the
**widest gap** of the three lines.

**3. Checkpoint × warmup confound.** `warmup_epochs=5`, LR ramped 2e-5 → 1e-4 over
ep1–5. The selected **best checkpoint is ep5 — exactly the end of warmup / the LR
peak**, immediately before overfitting set in. The tested model sits at a transient
LR point; LoRA never trained at the full, settled LR before its val peak. Also: the
safeguard (decline-based) never fired (val stayed flat near-peak), so the run was
**stopped and finalised manually** — best.pt's `model_config` was reconstructed and
the threshold set to **0.200** (the ep5 val-F1-optimal from the log) rather than via
the normal end-of-run protocol. Minor, but the threshold provenance is non-standard.

**4. N=1 and capacity untuned.** A single LoRA **rank (16)**, single α (16), single
LR (1e-4), single seed, single run. The "underfit" verdict could simply reflect
too-low rank — a rank sweep (32/64, or also adapting stage 3) is untested. No CV.

### Net
LoRA is **suggestively worse** (lower AUC, wider gap) than the unfreeze, but the
result is single-run, capacity-untuned, and selected at the warmup boundary. A fair
re-test needs a rank sweep, a checkpoint past warmup at settled LR, and CV/multi-seed
before calling LoRA a refuted direction.
